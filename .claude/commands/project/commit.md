# Commit Changes

## Instructions

1. Run `git status` to see changes
2. Run `git diff --stat` for overview
3. Stage relevant files (not .env, *.db, node_modules, target, __pycache__)
4. Create commit with format: `phase(N): description`
5. Examples:
   - `phase(1): project scaffolding and dependency setup`
   - `phase(8): implement scanner discovery phase`
   - `phase(21): implement neural node component with visual encoding`
   - `phase(31): integration tests and performance audit`
